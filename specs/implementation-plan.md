## Execution Key

- **`[P]`** — Task can run in parallel with other `[P]`-marked tasks in the same stage (different files, no shared mutable state, no dependencies between them)
- **`[E*-S*]`** — Traces to PRD epic story (e.g. `E1-S4` = Epic 1 Story 4, coverage ratchet)
- **`[ADR-NNN]`** — Task implements a decision from the referenced ADR
- **`[P0]` through `[P9]`** — Maps to `specs/typescript-rewrite-plan.md` §7 phase numbering
- **`Blocker`** — Task that must complete before the next milestone begins
- **`Checkpoint`** — Human approval gate before subsequent stage begins

## Milestone 3: M2 — Leaf + Config Clusters Ported (Stage 4.1)

**Goal:** Zero-dependency utility layer + core config layer ported to TypeScript. `bin/lib/config-yaml.cjs` eliminated. Hand-rolled YAML parser in `config-loader.js` deleted.

**Rewrite Plan phase:** P1 (leaf utilities) + P2 (schema & validation first half).

**Stories covered:** E3-S1, E3-S2.

**Per-module port recipe** (invariant across all E3 stories — augmented per Pit Crew QA Turn-2 finding for IPC v0/v1 fixture authoring at port time, not deferred to M8):

1. Pre-flight: `tests/test-<name>.test.js` passes on v1.1.14 baseline (recorded in PR description).
2. Create `bin/lib-ts/<name>.ts` with typed signatures; absorb existing logic; use typed errors from `src/errors.ts` in place of `process.exit()`.
3. Keep named-exports CJS-shape identical; `tsdown` emits `bin/lib-ts/<name>.js` + `.d.ts` to `dist/lib/` during build.
4. `tests/test-<name>.test.js` passes **unchanged** via strangler `paths` alias (resolves `@lib/<name>` → `bin/lib-ts/*` first, falls back to `bin/lib/*`).
5. Cross-module contract harness: zero new drift.
6. Coverage ratchet: at or above baseline.
7. CLI help snapshot: byte-identical pre/post port.
8. Holodeck baseline: PASS.
9. Per-file NFR-D02 (machine-readable return shapes): applicable functions have typed `@returns` OR TS return annotations.
10. Ship in next 1.x release (patch or minor per ADR-008 semver-discipline decision).
11. **IPC v0/v1 backward-compat fixture authoring** (per ADR-007, applied at port time NOT deferred to M8): for every IPC-eligible module, commit `tests/fixtures/ipc/<name>/v0/input.json` + `v0/expected-stdout.json` (envelope without `"version"` field — v1.1.14-shape) AND `tests/fixtures/ipc/<name>/v1/input.json` + `v1/expected-stdout.json` (envelope with `"version": 1`). Replay test (`tests/test-<name>-ipc-compat.test.ts`) pipes each `input.json` to `node dist/lib/<name>.js` and asserts byte-identical stdout against `expected-stdout.json`. Authoritative naming scheme: ADR-007's `<v>/input.json` + `<v>/expected-stdout.json` (PRD E4-S3's `in.json`/`out.json` references are superseded; PM-phase will be amended in a follow-up patch — captured in Deviation Log).
